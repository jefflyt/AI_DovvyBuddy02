# Step 1.2: Create Base TypeScript Configuration

**Part of:** Step 1 from plan.md  
**Focus:** Create root tsconfig.json with strict mode and path aliases  
**File to Create:** `tsconfig.json` (root)  
**Estimated Time:** 10 minutes

## Overview

This substep creates the base TypeScript configuration that all workspaces will extend. It enables strict type checking, configures path aliases for cross-workspace imports (`@shared/*`), and sets up modern ES module support.

## Pre-Substep Checklist

- [ ] **Branch:** Working on `1.1-monorepo-init`
- [ ] **Prior Step:** Completed Step 1.1 (package.json created, dependencies installed)
- [ ] **Verification:** `npx tsc --version` returns version 5.3.3+

## Implementation

### Goal
Create base tsconfig.json with strict mode, path aliases, and workspace-friendly settings.

### Step-by-Step Instructions

#### Step 1: Create Root tsconfig.json
- [ ] Ensure you're in the project root directory
- [ ] Copy and paste the following content into `tsconfig.json` at the root:

```json
{
  "compilerOptions": {
    // Type Checking
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "strictBindCallApply": true,
    "strictPropertyInitialization": true,
    "noImplicitThis": true,
    "alwaysStrict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    "noUncheckedIndexedAccess": true,

    // Modules
    "module": "ESNext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "allowJs": true,
    "checkJs": false,

    // Emit
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "outDir": "./dist",
    "removeComments": true,
    "noEmit": true,

    // Interop Constraints
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "forceConsistentCasingInFileNames": true,
    "isolatedModules": true,

    // Language and Environment
    "target": "ES2022",
    "lib": ["ES2022"],
    "jsx": "preserve",

    // Path Aliases (for cross-workspace imports)
    "baseUrl": ".",
    "paths": {
      "@shared/*": ["src/shared/*"],
      "@shared": ["src/shared/index.ts"]
    },

    // Skip type checking of declaration files
    "skipLibCheck": true
  },
  "exclude": [
    "node_modules",
    "dist",
    "build",
    ".next",
    "coverage"
  ]
}
```

#### Step 2: Verify TypeScript Configuration
- [ ] Run TypeScript compiler in check mode:

```bash
npx tsc --noEmit
```

Expected output: No errors (project is empty, so no files to check yet)

- [ ] Verify the configuration is valid:

```bash
npx tsc --showConfig
```

This should display the merged configuration without errors.

## Verification Checklist

- [ ] File saved in correct location: `tsconfig.json` (at project root)
- [ ] `npx tsc --noEmit` runs without errors
- [ ] `npx tsc --showConfig` displays configuration correctly
- [ ] Path aliases configured: `@shared/*` maps to `src/shared/*`
- [ ] Strict mode enabled (all strict* options set to true)

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `error TS5023: Unknown compiler option` | Ensure TypeScript 5.3+ is installed: `npx tsc --version` |
| `Cannot find module '@shared'` when testing later | This is expected until workspaces are created in Step 3 |
| JSON parse error | Validate JSON syntax at jsonlint.com or use VS Code JSON validation |
| Path resolution not working | Ensure `baseUrl` is set to `.` and `paths` uses correct relative paths |
